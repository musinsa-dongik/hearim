# 헤아림 데일리 템플릿

아래 형식을 따라 작성한다. 섹션 순서를 지킨다.

```markdown
## 헤아림 데일리: YYYY-MM-DD (요일)

### 오늘의 작업 (Context)

**이슈:** [오늘 하루를 관통하는 작업 맥락을 1-2문장으로 요약]

**해결:**

1. **[작업 제목]** (`커밋해시`): [무엇을 했는지 구체적으로]. 변경 파일과 규모를 명시한다(+N/-M줄).

2. **[작업 제목]** (`커밋해시`): [동일 구조로 작성]

> 커밋 해시와 변경량을 반드시 포함하여 나중에 추적 가능하게 한다.
> 여러 커밋이 하나의 작업 흐름이면 묶어서 기술한다.
> 코드 리뷰, 문서 작업, 커뮤니케이션 등 비개발 작업도 포함한다.

---

### 배움 카드 (Concepts)

#### 1. [개념/패턴 이름]: [오늘 경험과 연결한 부제]

[3-5문장으로 설명. "오늘 내가 왜 이렇게 판단했는가"를 중심으로 작성한다.]
[일반적 개념 설명이 아니라, 오늘의 구체적 상황에서 어떤 선택을 했고 왜 그랬는지를 서술한다.]
[판단 기준이 있었다면 명시하고, 반대로 선택하지 않은 대안과 그 이유도 언급한다.]

**패턴:**
```typescript
// Before: 변경 전 코드 (실제 구현 기반)
// After: 변경 후 코드 (실제 구현 기반)
```

**언제 쓰는가:**
- [적용 상황 -- 구체적 조건과 함께]
- [반대로 이 패턴이 맞지 않는 경우도 언급]

---

#### 2. [개념/패턴 이름]: [부제]

[동일 구조로 작성]

---

### 이전 학습 질의 연결

- **[이전 질의 주제]** (M/DD 질의): [오늘 어떻게 답을 찾았는지, 또는 부분적으로 해소된 내용]
- **[이전 질의 주제]** (M/DD 질의): [연결 설명]

> 이전 헤아림이 없으면 이 섹션을 생략한다.
> 이전 질의와 직접 연결되지 않더라도, 관련 주제가 오늘 등장했으면 언급한다.

### 추가 학습 질의

- [ ] [구체적 도구/패턴명을 포함한 질의 -- "다음에 뭘 해야 하는지"가 명확하도록]
- [ ] [비교 대상이 있으면 명시 (A vs B), 트레이드오프 관점의 질문]

---

### 튜터의 한마디

[메타인지적 관점에서 오늘의 학습을 정리. 성장 방향 제시. 구체적 다음 단계 제안 포함. 2-3문장.]
```

## 작성 기준

- 배움 카드: 최소 2개, 최대 4개
- 각 카드는 **"나의 맥락에서 배운 것"**을 기록 -- 교과서적 개념 설명이 아님
- 코드 예시는 실제 오늘 작성한 코드 기반으로 Before/After 비교
- 판단 기준, 트레이드오프, 반례(이 방법이 맞지 않는 경우)를 포함
- 커밋 해시와 변경량으로 작업의 추적 가능성 확보
- 이전 헤아림의 "추가 학습 질의"와 오늘 배운 것을 연결
- 추가 학습 질의는 구체적 도구명/패턴명을 포함하여 다음 행동이 명확하도록
- "헷갈린다", "추가 학습 필요" 표현 허용
- 이모지를 쓰지 않는다 (섹션 제목 포함)
- 작성 시간 목표: 10분 이내
